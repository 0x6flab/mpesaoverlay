# gRPC Adapter

Mpesaoverlay has a simple gRPC adapter that can be used to send messages to the
API using gRPC. This allows users who develop applications in languages that
have gRPC support to send messages to the API.

## Configuration

The gRPC adapter is configured using the `gRPC Adapter` section in the `docker/.env` file.

```env
### GRPC Adapter
MO_GRPC_HOST=
MO_GRPC_PORT=9000
MO_GRPC_URL=${MO_GRPC_HOST}:${MO_GRPC_PORT}
MO_GRPC_SERVER_CERT=
MO_GRPC_SERVER_KEY=
```

- `MO_GRPC_HOST` - The hostname of the gRPC adapter. It defaults to `localhost`.
- `MO_GRPC_PORT` - The port of the gRPC adapter. It defaults to `9000`.
- `MO_GRPC_URL` - The URL of the gRPC adapter. It defaults to `localhost:9000`.
- `MO_GRPC_SERVER_CERT` - The path to the server certificate. It defaults to empty.
- `MO_GRPC_SERVER_KEY` - The path to the server key. It defaults to empty.

## Running

To run the gRPC adapter, you can use the following command:

```bash
docker-compose up -d grpc
```

or if you want to run the binary directly:

```bash
make grpc
./build/mpesa-grpc
```

## Usage

The gRPC adapter is used by sending payload to the following endpoint:

- `mpesaoverlay.grpc.Service/Token` - GetToken
- `mpesaoverlay.grpc.Service/ExpressQuery` - ExpressQuery
- `mpesaoverlay.grpc.Service/ExpressSimulate` - ExpressSimulate
- `mpesaoverlay.grpc.Service/B2CPayment` - B2CPayment
- `mpesaoverlay.grpc.Service/AccountBalance` - AccountBalance
- `mpesaoverlay.grpc.Service/C2BRegisterURL` - C2BRegisterURL
- `mpesaoverlay.grpc.Service/C2BSimulate` - C2BSimulate
- `mpesaoverlay.grpc.Service/GenerateQR` - GenerateQR
- `mpesaoverlay.grpc.Service/Reverse` - Reverse
- `mpesaoverlay.grpc.Service/TransactionStatus` - TransactionStatus
- `mpesaoverlay.grpc.Service/RemitTax` - RemitTax
